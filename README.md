# Подписка для обхода белых списков от zieng2

Подписка предназначена для обхода ограничений мобильной связи. Прошу стараться применять её по назначению.

Все файлы имеют один источник и обновляются один раз в час.

Названия файлов кликабельны — используйте это, чтобы получить ссылку для добавления в клиент.

## Вариации подписки

Рекомендованная версия — [**vless_lite.txt**](https://raw.githubusercontent.com/zieng2/wl/main/vless_lite.txt). Она содержит только проверенные подсети, протестированные во время ограничений, и подходит практически для всех клиентов.

Остальные варианты:

+ [**vless_universal.txt**](https://raw.githubusercontent.com/zieng2/wl/main/vless_universal.txt) — полная версия подписки. Содержит все доступные подсети.

+ [**vless.txt**](https://raw.githubusercontent.com/zieng2/wl/main/vless.txt) — устаревшая версия подписки. **Поддержка прекращена**.

## Как пользоваться?

Используя список подписок выше, выберите подходящую для вас версию, скопируйте ссылку и добавьте её в клиент. Обычно это можно сделать с помощью кнопки «Добавить из буфера обмена» или вручную через кнопку «Добавить».

Важно: копировать и добавлять в клиент нужно саму ссылку на подписку. Вручную копировать её содержимое неправильно.

После загрузки подписки в клиент рекомендуется включить её автообновление. Подписка обновляется каждый час не просто так — профили быстро теряют актуальность. Работающие сегодня серверы могут уже не работать завтра, поэтому, чтобы не обновлять подписку вручную, настроим автоматизацию. Для этого перейдите в настройки подписки и включите автообновление, выбрав подходящий период. В части клиентов эта функция находится в общих настройках. Как правило, обновления раз в 24 часа должно быть достаточно, однако вы можете настроить интервал по своему усмотрению, исходя из опыта использования.

Следующим шагом перейдите в настройки клиента и включите опцию «Разрешить небезопасные». В некоторых клиентах такой глобальной опции нет — в этом случае её можно найти в настройках конкретной подписки. Если подобная опция отсутствует полностью, дополнительных действий не требуется. Включение этой опции позволяет увеличить количество доступных серверов.

На данном этапе вы уже имеете полностью работоспособный клиент с добавленной подпиской. Как же понять, к какому серверу подключаться во время ограничений связи?

Главный инструмент, которым вы будете пользоваться, — это проверка «задержки» до сервера сразу для всей подписки. Не путать с TCP/ICMP-пингом, первый из которых не отражает доступность сервера для полноценного подключения при ограничениях связи, а второй может не работать вовсе в зависимости от настроек серверов.

В разных клиентах запуск проверки может отличаться по названию и расположению (URL Test, Latency Test и т.п.), однако такая функция присутствует обязательно. После завершения проверки отобразится список доступных серверов. В большинстве клиентов доступна сортировка по задержке — используйте её, чтобы выбрать наиболее быстрый сервер. Подключитесь к подходящему для вас — и вы получите доступ к интернету.

_Замечание:_ на большей части российских серверов в подписке доступны заблокированные сервисы, подключаться к зарубежным профилям с такой целью необходимости нет.

## Советы и полезная информация

**1.** В зависимости от используемого клиента список доступных серверов может отличаться. Рекомендуется попробовать разные клиенты.

**2.** DNS в настройках клиента не должен влиять на работоспособность подключения. Ранее наблюдалась проблема: при недоступности DNS-сервера профили, содержащие доменные имена, не работали до смены DNS (например, на Яндекс DNS). В подписке такие конфигурации по-прежнему присутствуют, однако в актуальной версии они используют соответствующие IP-адреса, что исключает зависимость от доступности DNS-сервера.

**3.** Вне ограничений связи должна быть доступна большая часть серверов. Работоспособность конфигураций проверяется скриптом при обновлении.

**4.** Формирование подписки осуществляется на основе публичных и непубличных источников.
