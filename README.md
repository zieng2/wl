# Подписка для обхода белых списков от zieng2

Подписка предназначена для обхода ограничений мобильной связи, но также может использоваться и при доступном интернете. В таком случае рекомендуется использовать версию [**vless_universal.txt**](https://raw.githubusercontent.com/zieng2/wl/main/vless_universal.txt). Тем не менее, прошу стараться использовать подписку по назначению.

Все четыре файла имеют один источник и обновляются одновременно.

Названия файлов кликабельны — используйте это для получения ссылки для добавления в клиент.

## Вариации подписки

Рекомендованная версия — [**vless_lite.txt**](https://raw.githubusercontent.com/zieng2/wl/main/vless_lite.txt). Она содержит только проверенные подсети, протестированные во время ограничений, и подходит практически для всех клиентов. 

Остальные варианты:

+ [**vless_universal.txt**](https://raw.githubusercontent.com/zieng2/wl/main/vless_universal.txt) — полная версия подписки. Содержит все доступные подсети.

+ [**vless_nolite.txt**](https://raw.githubusercontent.com/zieng2/wl/main/vless_nolite.txt) — используется исключительно для тестирования. Содержит все подсети, кроме проверенно рабочих.

+ [**vless.txt**](https://raw.githubusercontent.com/zieng2/wl/main/vless.txt) — устаревшая версия подписки, не добавляющаяся в v2rayN. Использовать **не рекомендуется**.

## Советы и полезная информация

**1.** Рекомендуется глобально включить опцию 'Разрешить небезопасные' в клиенте.

**2.** По отзывам пользователей, в зависимости от клиента могут отличаться рабочие сервера. Попробуйте разные клиенты.

**3.** Формирование подписки осуществляется на основе различных публичных и непубличных источников.

**4.** DNS в настройках клиента не должен влиять на работоспособность. Ранее была замечена проблема, что из-за недоступности DNS от Cloudflare / Google / других, конфиги с доменными именами не работали до смены на Яндекс. Те конфиги все еще присутствуют в подписке, но в актуальной версии подписки все они сразу представлены с соответствующими IP-адресами.

**5.** При доступном интернете должны работать вплоть до 100% серверов. Скрипт проверяет работоспособность конфигов при обновлениях.
